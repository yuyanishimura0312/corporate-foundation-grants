# 研究奨励金・学会賞・論文賞・技術賞を運営する財団調査

調査日: 2026-05-08  
対象カテゴリ: (p) 研究奨励金・学会賞を運営する財団  
目的: 研究助成財団DBのカバレッジ補完。単なる公募型研究助成だけでなく、顕彰、研究奨励金、博士論文賞、技術賞、助成成果表彰を持つ団体を抽出。

## 1. 重要所見

- DB補完優先度が高いのは、`研究助成 + 顕彰` を同一財団内で運営する団体。例: 東レ科学振興会、NEC C&C財団、立石科学技術振興財団、市村清新技術財団、井上科学振興財団、内藤記念科学振興財団、武田科学振興財団、上原記念生命科学財団。
- 「賞」は研究費DBで漏れやすいが、副賞が研究費・研究奨励金として支給されるものが多い。例: 井上春成賞は研究者へ研究奨励金、電気科学技術奨励賞は助成金付き、風戸研究奨励賞は研究費助成付き。
- 年間助成額は、財団により公開粒度が異なる。事業報告・募集要項で総額が明記されるもの、個別賞金のみ明記されるもの、採択件数だけ公開されるものが混在する。DBでは `annual_program_budget`, `award_amount_per_case`, `budget_disclosure_level` を分けるのが安全。
- 親テーマ上のタグは複数付与が必要。例: NEC C&C財団は `(e)情報通信・電機メーカー系` かつ `(p)賞`; 武田・上原・内藤・持田は `(f)製薬・医療機器メーカー系` かつ `(p)賞`; 花王・東レ・近藤は `(b)化学・素材` かつ `(p)賞`。

## 4. 推奨スキーマ追加

既存DBが研究助成中心の場合、賞を取り込むため以下の追加フィールドが必要。

| フィールド | 型 | 用途 |
|---|---|---|
| `program_type` | enum | `research_grant`, `award`, `young_investigator_award`, `paper_award`, `technology_award`, `travel_award`, `publication_award` |
| `award_amount_per_case` | integer | 1件あたり賞金・副賞・研究奨励金 |
| `annual_program_budget` | integer/null | 年間総額。未公表ならNULL |
| `budget_disclosure_level` | enum | `total_disclosed`, `per_case_only`, `not_disclosed`, `inferred_from_cases` |
| `nomination_type` | enum | `open`, `institutional_nomination`, `society_nomination`, `closed_nomination`, `foundation_selection` |
| `award_basis` | enum | `research_achievement`, `doctoral_thesis`, `paper`, `technology_commercialization`, `grant_output`, `young_researcher_potential` |
| `related_parent_category` | text[] | 親テーマ(a)-(q)の複数タグ |
| `latest_awardees_url` | text | 受賞者一覧・採択実績ページ |

## 5. 優先取り込み順
