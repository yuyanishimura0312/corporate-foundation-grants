## 7. 優先度付き改善提案（カバー率向上に効く順）

### Priority A（最大のリフト・1〜2週間で実装可能）

1. **Grant DB から財団系485件を取り込む**
   - 名寄せキーは「正規化名（接頭辞・空白・全角半角を除去）」。取込時に corporate_parent を企業キーワード辞書から自動推定。
   - 期待効果: 189件 → **約670件（約3.5倍）**。一回の作業で母集団が大きく拡張。

2. **重複7組の名前マージ**
   - 「公益財団法人 + ベース名」を正準形に統一し、ID統合。grant_programs / grant_calls の FK は移行スクリプトで貼替え。
   - 期待効果: 即時にデータ品質向上。

3. **JFC TOP300 の rank と annual_grant_amount を一括投入**
   - JFC 101〜300 が現在 0件。少なくとも上位300財団の名前と金額は外部資料から表で取れる。
   - 期待効果: ランキング検索・「金額順並べ替え」が機能するようになる。

### Priority B（収集パイプラインに組み込む・1ヶ月）

4. **organizations の `prefecture` `url` `contact_address` を法人登記情報からバルク補完**
   - 内閣府の公益法人インフォメーション（公開API）または国税庁法人番号公表サイトから所在地・URLを取得。
   - 期待効果: 都道府県NULL 92件 → 大半補完。都道府県別検索が機能。

5. **grant_calls の `fiscal_year` `application_deadline` `grant_amount_max` を年次スクレイプで蓄積**
   - 各財団の公募ページを年1回（4月）スナップショット保存。FYと締切の欠損率を半減目標。
   - 期待効果: 89.9% → 50%以下を目標。「過去3年の公募」分析が可能に。

### Priority C（中期・データ拡張）

6. **`grant_programs.subcategories` の埋め込み（研究分野の細分化）**
   - description テキストから KAKEN 分野コードまたは独自タクソノミーで自動分類。
   - 期待効果: 「ライフサイエンス系の研究助成だけ見たい」などのフィルタが可能。

7. **`grant_calls.expected_awards` `actual_awards` の事業年報からの補完**
   - 公益法人の事業報告書PDFから採択件数を抽出（既存 pdf_pipeline 流用）。
   - 期待効果: 採択倍率の見える化。

8. **`name_en` の英語名併記（93%欠損）**
   - 国際比較・海外向けポータル展開時の前提整備。優先度は低いが将来のために。
